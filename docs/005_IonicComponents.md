_Última atualização do doc. 09.Abr.2026_

> Introdução aos **componentes do Ionic**, descrevendo os **elementos de interface de utilizador (UI)** pré-construídos e personalizáveis que seguem os padrões visuais do **Material Design** e do **iOS**. Explica as vantagens de utilizar estes componentes - como a **familiaridade visual**, tornando uma app mais intuitiva, e a **aceleração do desenvolvimento** - e identifica as **secções principais da documentação oficial**, como *Basic Usage*, *Properties* e *CSS Custom Properties*, essenciais para compreender e personalizar cada componente.


# Notas gerais sobre os componentes

- [UI Components - Documentação](https://ionicframework.com/docs/components)

- São elementos de UI (_User Interface_) pré-construídos e personalizados;

- São personalizáveis;

- São construídos com base nos padrões de design do Material Design (Google) e do iOS (Apple). Consequentemente estão preparados para terem o aspeto nativo em ambos os sistemas operativos; 

- A aparência e a UX (_User eXperience_) é familiar e, consequentemente, intuitiva para os utilizadores de dispositivos Android e iOS;

- Permite que o programador se dedique à lógica de negócio e nas funcionalidades, permitindo, desta forma, acelerar o processo de desenvolvimento de uma app;

- Exemplos de componentes:
    - Botões;
    - Listas;
    - _Cards_;
    - Alertas;
    - Modais;
    - Formulários;
    - Menus;
    - (…)
